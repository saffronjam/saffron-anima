# Phase 9 — The reproducible-gate orchestration

**Status:** COMPLETED

**Depends on:** 13-testing-and-verification:phase-2-golden-snapshot-infrastructure, 13-testing-and-verification:phase-3-bun-e2e-as-parity-harness, 13-testing-and-verification:phase-5-validation-clean-gate, 13-testing-and-verification:phase-6-control-schema-contract-gate, 13-testing-and-verification:phase-8-self-test-removal-ledger, 01-build-and-toolchain:phase-6-reproducible-gate

## Goal

Tie every test layer into one sequenced gate — the `tools/ci/check.sh` / `make check` equivalent — so a
single command reproduces the full verification and a regression in any layer fails the whole thing.
This is the test-architecture counterpart to `01-build-and-toolchain` phase 6 (which owns the justfile +
toolbox env); this phase owns *what the gate runs and in what order*, and asserts the four standing
gates + the self-test-removal grep are all wired into it.

The orchestrated sequence (mirroring `check.sh`'s `step` blocks, adapted to Cargo):

1. **Workspace build** — `cargo build --workspace` (replaces the CMake/ninja engine build; the
   `slangc`/`cxx` build scripts run inside it).
2. **Codegen freshness** — regenerate `@saffron/protocol` + OpenRPC + manifest from the Rust DTOs and
   `git diff --exit-code` the generated files (replaces `check.sh:22`'s `gen.ts` + diff step; the
   generated files now come from `10-protocol-codegen`'s `xtask`). Plus the Luau-typegen freshness
   check (`12-scripting` phase 9) that replaces the deleted `check-script-defs`.
3. **Unit + crate tests** — `cargo test --workspace` (the inline `#[cfg(test)]` + crate `tests/` suites,
   including every ported self-test oracle and the golden snapshot tests).
4. **Self-test-removal assertion** — the phase-8 grep: no `run*SelfTest` / `SAFFRON_SELFTEST` /
   `fn *self_test` outside `#[cfg(test)]`.
5. **Present-only smoke + validation-clean** — `SAFFRON_EXIT_AFTER_FRAMES=5` on the Rust binary, log
   grepped for `[saffron:vulkan] error: [validation]` (phase 5).
6. **Control-schema contract** — `check-control-schema/check.ts` against the Rust binary (phase 6).
7. **Project startup smoke** — the `check-projects` equivalent (Rust engine boots a project, asset
   layout sane).
8. **e2e** — the bun suite (or its enabled slice) against the Rust binary (phase 3).
9. **Frontend** — `bun run build` + `bun test` in `editor/` (unchanged; proves the editor compiles
   against the regenerated protocol).
10. **Determinism gate** — when present, the physics cross-arch trace gate
    (`05-physics-jolt-bridge` phase 5) run as a standing check.

Each step fails the gate independently (the `fail=1` accumulation pattern from `check.sh`), and the gate
prints a per-step pass/fail summary.

## Why this shape (NO LEGACY)

- **One gate, mirroring the one the C++ had.** `tools/ci/check.sh` was the single reproducible gate; the
  Rust gate is its direct successor with Cargo replacing CMake and `cargo test` replacing the in-engine
  self-tests. There is one gate command, not a "quick" and a "full" pair — the conventions allow one way
  to verify. The `make check` lore (toolbox, headless weston, NVIDIA ICD env) is carried verbatim into
  the justfile by `01-build-and-toolchain`; this phase only sequences the checks.
- **The four standing gates are *in* the gate, not adjacent to it.** The feasibility study's whole point
  is that validation-clean, the contract test, determinism, and golden/snapshot are "the only automated
  detectors for the entire silent-failure class" and must be "first-class, continuously-green." Wiring
  them into the one reproducible gate is what makes "continuously green" enforceable rather than
  aspirational — a drift in any one turns the gate red.
- **Codegen freshness replaces the C++ generated-file diff and the script-defs drift check together.**
  `check.sh` had both a `gen.ts`-output diff (`:24`) and a separate `check-script-defs` step (`:45`).
  Under the rewrite the protocol artifacts are generated by `xtask` and the Luau defs by the single
  binding source (PP-8) — so the freshness check is "regenerate and diff," and the hand-synced
  `library/sa.lua` drift tripwire is *deleted*, replaced by a generated-defs-are-up-to-date diff. One
  freshness check, no hand-synced second copy.

## Acceptance gate

- A single `just check` (or the toolbox-wrapped equivalent) runs all ten steps in order against the Rust
  binary, accumulates failures, and prints a per-step pass/fail summary ending in a clear ALL/SOME
  verdict (the `check.sh` shape).
- The four standing gates are each invoked by the orchestrator: validation-clean (step 5),
  control-schema contract (step 6), golden/snapshot (inside step 3), determinism (step 10 when present).
- The self-test-removal grep (step 4) is wired and green.
- The codegen-freshness diff (step 2) is green: regenerating the protocol + Luau defs leaves no `git
  diff` (proves the committed generated artifacts match the Rust source).
- The full gate is green on the walking-skeleton milestone (a minimal but real Rust engine), and stays
  green as later phases extend it; the Cargo workspace compiles.

## Audit result (gate)

The orchestrator is `tools/ci/check.sh` (invoked by `just check` and `.github/workflows/ci.yml`),
rewritten as the Cargo/xtask/bun successor of the C++ gate. It sequences the ten steps in the
canonical order above, accumulates failures, and prints a per-step `PASS`/`FAIL`/`DEFER` verdict plus
an `ALL GATES PASSED` / `SOME GATES FAILED` line. The `defer` machinery for not-yet-built phases is
retired (NO LEGACY): the host is real and `xtask gen-protocol` exists, so every live-engine step is now
a hard gate; `defer` survives only for genuinely host-absent tools (bun off PATH).

## Hardware/display-gated items this gate cannot fully run here (documented, not silently skipped)

The toolbox is x86_64 with a software GPU (llvmpipe), so two legs run only on the self-hosted runner:

- **The determinism gate's aarch64 leg.** Step 3 runs the gate's x86 half (the SHA-256 trace hash is
  stable run-to-run and build-to-build, flags confirmed active). The non-negotiable
  `Rust-x86 hash == Rust-aarch64 hash` assertion is `DEFERRED-NEEDS-HARDWARE` in
  `physics/tests/determinism.rs` until the self-hosted aarch64 runner re-derives the trace.
- **The cross-engine parity rig** (`tests/e2e/parity.test.ts`, phase 7) records two **measured
  tolerances** rather than asserting byte-identity: the `project.json` raw-byte key order (data
  round-trips correctly) and the studio-lit preview render under llvmpipe (a lighting/tonemap delta,
  not geometry — the byte-exact preview match is meaningful only on the editor's real GPU). These are
  sign-off inputs for `14-migration` / the #106 parity sign-off, not gate failures.
